## `getFlag(key)`

Retorna os dados de uma flag específica, ou `undefined` se a flag não existir.

```ts
const flag = canary.getFlag('new-checkout')
// FlagData | undefined
```

### Verificação segura com nullish coalescing

```ts
const isEnabled = canary.getFlag('new-checkout')?.enabled ?? false
```

<Callout type="warning">
  O SDK não expõe um método `isEnabled()`. Use sempre o padrão
  `getFlag(key)?.enabled ?? false`.
</Callout>

## `getFlags()`

Retorna todas as flags do environment como array.

```ts
const flags = canary.getFlags()
// FlagData[]

for (const flag of flags) {
  console.log(flag.key, flag.enabled)
}
```

## Exemplos por tipo de flag

### Boolean flag

```ts
const maintenance = canary.getFlag('maintenance-mode')

if (maintenance?.enabled) {
  return <MaintenancePage />
}
```

### Rollout flag

```ts
const newDashboard = canary.getFlag('new-dashboard')

// flag.enabled é true ou false baseado no userId e percentage
if (newDashboard?.enabled) {
  return <NewDashboard />
}

// Para ver a porcentagem configurada:
if (newDashboard?.type === 'rollout') {
  console.log(`${newDashboard.percentage}% dos usuários verão esta feature`)
}
```

## Antes do `init()`

Se você chamar `getFlag()` antes de `await init()`, a flag retornará `undefined` (o cache local ainda está vazio).

```ts
const client = new CanaryGate('cg_key')
// Não faça isso:
client.getFlag('feature') // undefined!

// Faça isso:
await client.init()
client.getFlag('feature') // FlagData | undefined ✓
```

## Flags que não existem

Se a flag não existir no environment, `getFlag()` retorna `undefined`. Sempre use `?? false` como fallback para não quebrar o fluxo da aplicação:

```ts
// Se 'new-feature' não existe no environment, isEnabled será false
const isEnabled = canary.getFlag('new-feature')?.enabled ?? false
```
