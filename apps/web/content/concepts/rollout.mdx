## Como o rollout funciona

Quando uma flag é do tipo `rollout`, o CanaryGate calcula se um usuário específico deve ver a feature baseado em:

1. O **userId** do usuário (passado nas options do SDK ou implícito)
2. A **chave da flag** (`key`)
3. A **porcentagem** configurada no painel

O cálculo é um hash determinístico: o mesmo usuário com o mesmo userId sempre receberá o mesmo resultado para a mesma porcentagem. Isso garante **stickiness** — o usuário não fica alternando entre versões.

## Configurando no SDK

```ts
const canary = new CanaryGate('cg_api_key', {
  userId: currentUser.id // ou string anônima estável
})
await canary.init()

const flag = canary.getFlag('new-dashboard')
if (flag?.enabled) {
  // este usuário está no rollout
}
```

<Callout type="info">
  Se você não passar `userId`, o SDK usa um ID anônimo estável gerado e
  persistido em `localStorage` (browser) ou em memória (Node.js). Usuários
  anônimos são tratados de forma determinística também.
</Callout>

## Aumentando a porcentagem

O workflow típico de canary deploy:

| Fase     | Porcentagem | Ação                          |
| -------- | ----------- | ----------------------------- |
| Beta     | 5%          | Monitorar métricas e erros    |
| Expansão | 25%         | Confirmar estabilidade        |
| Ramp     | 50%         | Metade do tráfego             |
| GA       | 100%        | Feature disponível para todos |

Cada ajuste de porcentagem é entregue em tempo real via SSE para todos os clientes conectados.

## Rollout manual vs automático

- **Manual**: Você ajusta a porcentagem manualmente pelo painel
- **Automático** (plano Pro): Configure um schedule que aumenta a porcentagem automaticamente em horários definidos

Veja [Schedule e Auto-rollout](/docs/concepts/schedule-auto-rollout) para mais detalhes.
