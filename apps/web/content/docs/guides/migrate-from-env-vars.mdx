title: Migrar de Variáveis de Ambiente
description: Como substituir feature toggles baseados em env vars pelo CanaryGate.

## O problema com env vars para feature flags

É comum começar usando variáveis de ambiente como flags:

```ts
// ❌ Abordagem com env vars
if (process.env.ENABLE_NEW_CHECKOUT === 'true') {
  // novo checkout
}
```

Isso funciona para começar, mas traz problemas:

- **Redesploy obrigatório** para toda mudança de feature
- **Sem histórico** de quem mudou o quê e quando
- **Sem rollout gradual** — é tudo ou nada
- **Sem reatividade** — o código não atualiza sem reiniciar

## Estratégia de migração

### 1. Instale o SDK

```bash
npm install @canarygate/js
```

### 2. Crie as flags no painel

Para cada `process.env.ENABLE_*` que você usa como toggle, crie uma flag Boolean equivalente no CanaryGate com o mesmo estado atual.

### 3. Migre um por um (sem big bang)

Crie um wrapper que lê de ambas as fontes durante a transição:

```ts
import { CanaryGate } from '@canarygate/js'

const client = new CanaryGate(process.env.CANARYGATE_KEY!)
await client.init()

function isFeatureEnabled(key: string, envFallback?: string): boolean {
  // Tenta o CanaryGate primeiro
  const flag = client.getFlag(key)
  if (flag !== undefined) return flag.enabled

  // Fallback para env var durante a transição
  if (envFallback) return process.env[envFallback] === 'true'

  return false
}
```

### 4. Substitua progressivamente

Antes:

```ts
if (process.env.ENABLE_NEW_CHECKOUT === 'true') {
```

Depois (fase de transição):

```ts
if (isFeatureEnabled('new-checkout', 'ENABLE_NEW_CHECKOUT')) {
```

Depois (migrado completamente):

```ts
if (client.getFlag('new-checkout')?.enabled ?? false) {
```

### 5. Remova as env vars

Quando todas as flags estiverem no CanaryGate e o wrapper não for mais necessário, remova as env vars do `.env` e dos secrets de CI/CD.

## Tabela de equivalências

| Padrão env var         | Equivalente CanaryGate        |
| ---------------------- | ----------------------------- |
| `ENABLE_X=true`        | Flag Boolean, enabled = true  |
| `ENABLE_X=false`       | Flag Boolean, enabled = false |
| `FEATURE_X_ROLLOUT=25` | Flag Rollout, percentage = 25 |

## Vantagens após a migração

- Mude features **sem redesploiar** — ideal para hotfixes e kill switches
- **Histórico completo** de quem ativou o quê e quando
- **Rollout gradual** nativo para features de risco
- **SSE em tempo real** — o SDK atualiza sem polling ou restart
