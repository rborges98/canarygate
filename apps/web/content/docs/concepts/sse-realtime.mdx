---
title: SSE e Tempo Real
description: Como o CanaryGate entrega atualizações de flags em tempo real via Server-Sent Events.
---

## Como funciona

Quando você chama `canary.init()`, o SDK abre uma conexão **SSE (Server-Sent Events)** com a API do CanaryGate. Essa conexão permanece aberta e recebe eventos automaticamente quando flags mudam.

```
SDK → init() → HTTP fetch (carga inicial de flags)
             ↓
          SSE stream (conexão persistente)
             ↓
     mudança no painel → evento SSE → SDK atualiza cache local
```

## Do painel ao cliente: latência

Quando você muda uma flag no painel:

1. A mudança é salva no banco de dados
2. Um evento é publicado no Redis Pub/Sub
3. O servidor SSE recebe o evento e transmite para todos os clientes conectados
4. O SDK do cliente recebe e atualiza o cache local

Latência típica: **< 500ms** da mudança no painel até o cliente receber.

## Reconexão automática

O `EventSource` nativo reconecta automaticamente em caso de queda de rede. O SDK também detecta quando a conexão expira e reconecta — você não precisa gerenciar isso manualmente.

## Verificando se os dados estão frescos

```ts
// Verifica se os dados podem estar desatualizados
if (canary.isStale()) {
  // A conexão SSE foi interrompida por mais de X tempo
  // Você pode recarregar ou mostrar um aviso ao usuário
}

// Última sincronização bem-sucedida
const lastSync = canary.getLastSyncAt()
console.log('Dados sincronizados em:', lastSync)
```

## Comportamento offline

Se a conexão SSE for perdida:

- As flags retornam os **últimos valores conhecidos** (cache local)
- `isStale()` retorna `true` após o timeout configurado
- A reconexão é automática quando a rede voltar

## Desconectando

```ts
// Importante: desconecte quando não precisar mais do SDK
// (ex: desmontagem de componente, shutdown do servidor)
canary.disconnect()
```

<Callout type="info">
  Em ambientes de teste ou scripts one-shot, chame `disconnect()` logo após
  `init()` para evitar que o processo fique aberto esperando eventos SSE.
</Callout>
