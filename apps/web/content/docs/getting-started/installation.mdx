## Requisitos

- **Node.js 18+** ou browser moderno (Chrome, Firefox, Safari, Edge atuais)
- **TypeScript 5+** (opcional, mas totalmente tipado)
- O SDK usa `EventSource` internamente para SSE — disponível nativamente em browsers e no Node.js 18+

## Instalação

```bash
npm install @canarygate/js
```

```bash
pnpm add @canarygate/js
```

```bash
yarn add @canarygate/js
```

## Uso com bundlers (Vite, Webpack, esbuild)

O pacote é publicado como ESM + CJS com declarações de tipos incluídas. Não requer configuração extra.

```ts
import { CanaryGate } from '@canarygate/js'
```

## Uso com Next.js

Em Next.js App Router, o SDK deve ser inicializado em um **Client Component** ou em código server-side que rode apenas uma vez (e.g., singleton).

```tsx filename="lib/canary.ts"
import { CanaryGate } from '@canarygate/js'

let instance: CanaryGate | null = null

export function getCanaryGate() {
  if (!instance) {
    instance = new CanaryGate(process.env.NEXT_PUBLIC_CANARYGATE_KEY!)
  }
  return instance
}
```

<Callout type="warning">
  Não inicialize o SDK em Server Components — ele abre uma conexão SSE que
  precisa de runtime de browser ou Node.js com suporte a `EventSource`.
</Callout>
