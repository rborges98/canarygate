## O que é um canary deploy

Um **canary deploy** é a prática de expor uma nova versão do software para uma pequena porcentagem de usuários antes de fazer o rollout completo. Se algo der errado, você reverte instantaneamente — sem redesploiar.

Com o CanaryGate, você faz canary deploys no nível de feature, não de infraestrutura.

## Configurando o canary

### 1. Crie uma rollout flag

```ts
// No painel: New Flag → Type: Rollout
// key: "v2-engine"
```

### 2. Inicialize com o userId

```ts
const client = new CanaryGate(process.env.CANARYGATE_KEY!, {
  userId: currentUser.id // ID estável — mesmo usuário sempre vê o mesmo resultado
})
await client.init()
```

### 3. Rampeie gradualmente

| Fase           | %    | Duração sugerida       |
| -------------- | ---- | ---------------------- |
| Canary inicial | 5%   | 1-2h monitorando erros |
| Expansão       | 20%  | 4-8h                   |
| Ramp           | 50%  | 24h                    |
| GA             | 100% | —                      |

Ajuste a porcentagem no painel. A mudança chega para os clientes em tempo real.

### 4. Avalie no código

```ts
const useV2 = client.getFlag('v2-engine')?.enabled ?? false

if (useV2) {
  return await processWithV2Engine(data)
} else {
  return await processWithV1Engine(data)
}
```

## Rollback instantâneo

Se você detectar um problema, volte a porcentagem para 0% no painel — sem redesploiar, sem downtime.

## Com auto-rollout (plano Pro)

Configure o aumento automático de porcentagem com um schedule:

```
10:00 → 5%
14:00 → 25%
amanhã 09:00 → 50%
48h depois → 100%
```

O CanaryGate executa cada etapa automaticamente. Você ainda pode pausar ou reverter manualmente a qualquer momento.

## Limpeza

Quando a feature estiver em 100% e estável, remova a flag do código e delete-a no painel para manter o projeto organizado.

```ts
// Antes (com flag):
const isNewEngine = client.getFlag('v2-engine')?.enabled ?? false
if (isNewEngine) { ... }

// Depois (100% rollout, flag removida):
// Código permanente sem condicional
```
