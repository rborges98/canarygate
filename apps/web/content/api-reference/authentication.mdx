## API Keys

A API do CanaryGate usa **API Keys por environment**. Cada environment de cada projeto tem sua própria key, criada em:

**Painel → Projeto → Environment → Settings → API Keys**

### Formato

```
cg_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## Enviando a API Key

### Header (recomendado)

```http
GET /api/v1/flags
Authorization: Bearer cg_live_your_key_here
```

### Exemplo com fetch

```ts
const response = await fetch('https://api.canarygate.io/api/v1/flags', {
  headers: {
    Authorization: `Bearer ${process.env.CANARYGATE_KEY}`,
    'Content-Type': 'application/json'
  }
})
```

### Exemplo com curl

```bash
curl https://api.canarygate.io/api/v1/flags \
  -H "Authorization: Bearer cg_live_your_key_here"
```

## Erros de autenticação

| Status | Código         | Descrição                                          |
| ------ | -------------- | -------------------------------------------------- |
| `401`  | `UNAUTHORIZED` | API Key ausente ou inválida                        |
| `403`  | `FORBIDDEN`    | API Key válida mas sem permissão para este recurso |

```json
{
  "error": "UNAUTHORIZED",
  "message": "Invalid or missing API key"
}
```

## Segurança

<Callout type="warning">
  Nunca exponha sua API Key no código do browser ou em repositórios públicos.
  Use variáveis de ambiente e, para browsers, use `NEXT_PUBLIC_` apenas com keys
  de environments públicos (staging/preview).
</Callout>

Para aplicações browser, considere criar uma **API Key read-only** separada para produção.
