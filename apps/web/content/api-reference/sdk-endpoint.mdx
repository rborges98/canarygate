O SDK usa dois endpoints especiais que não são voltados para uso direto, mas estão documentados aqui para fins de debugging e integrações avançadas.

## Carga inicial de flags

```http
GET /sdk/flags
Authorization: Bearer cg_live_your_key
```

Retorna todas as flags do environment em formato otimizado para o SDK, incluindo dados de rollout necessários para o hashing local.

## Stream SSE

```http
GET /sdk/events
Authorization: Bearer cg_live_your_key
Accept: text/event-stream
```

Abre uma conexão SSE. Eventos emitidos:

### `flags:updated`

Emitido quando qualquer flag do environment é criada, atualizada ou deletada.

```
event: flags:updated
data: {"key":"new-checkout","type":"boolean","enabled":true}
```

### `ping`

Emitido periodicamente para manter a conexão ativa.

```
event: ping
data: {}
```

## Usando manualmente (sem o SDK)

```ts
const source = new EventSource('https://api.canarygate.io/sdk/events', {
  // EventSource padrão não suporta headers — use um polyfill
  // ou o SDK que já lida com isso internamente
})

source.addEventListener('flags:updated', (event) => {
  const flag = JSON.parse(event.data)
  console.log('Flag updated:', flag)
})
```

<Callout type="warning">
  O `EventSource` nativo do browser não suporta envio de headers de
  autenticação. O SDK contorna isso internamente. Para uso manual, use uma
  biblioteca como `eventsource` (npm) que suporta headers.
</Callout>
